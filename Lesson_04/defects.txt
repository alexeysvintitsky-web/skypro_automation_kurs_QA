БАГ-РЕПОРТ №1

ID:              DEF-001
Заголовок:       Метод trim() удаляет только символ пробела, игнорируя другие пробельные символы
Приоритет:       Medium
Серьёзность:     Minor
Компонент:       StringUtils.trim()
Автор:           Svintitskiy A.
Дата:            2026-04-14

Описание:
Метод trim() удаляет только символ пробела (" "), но не удаляет другие
пробельные символы (табуляция "\t", неразрывный пробел и т.д.).

Шаги воспроизведения:
1. Создать экземпляр StringUtils
2. Вызвать метод trim() с аргументом "\tskypro"
3. Проверить результат

Ожидаемый результат: "skypro" (все пробельные символы удалены)
Фактический результат: "\tskypro" (табуляция не удалена)

Окружение: Python 3.14.3

Возможное решение:
Изменить реализацию метода, используя string.lstrip() или проверять все пробельные символы:
    def trim(self, string: str) -> str:
        return string.lstrip()



БАГ-РЕПОРТ №2

ID:              DEF-002
Заголовок:       Метод contains() выбрасывает исключение при поиске пустой строки
Приоритет:       Low
Серьёзность:     Minor
Компонент:       StringUtils.contains()
Автор:           Svintitskiy A.
Дата:            2026-04-14

Описание:
При передаче пустой строки ("") в качестве искомого символа в метод contains(),
возникает исключение ValueError, так как метод index() не может найти пустую строку.

Шаги воспроизведения:
1. Создать экземпляр StringUtils
2. Вызвать метод contains("hello", "")
3. Проверить результат

Ожидаемый результат: True (пустая строка содержится в любой строке) или False
Фактический результат: ValueError

Окружение: Python 3.14.3

Возможное решение:
Добавить проверку в начале метода:
    def contains(self, string: str, symbol: str) -> bool:
        if symbol == "":
            return True  # или False, в зависимости от требований
        return symbol in string



БАГ-РЕПОРТ №3

ID:              DEF-003
Заголовок:       Отсутствует обработка None во всех методах
Приоритет:       High
Серьёзность:     Major
Компонент:       StringUtils (все методы)
Автор:           Svintitskiy A.
Дата:            2026-04-14

Описание:
Методы класса не обрабатывают случай, когда вместо строки передаётся None.
Это приводит к исключению AttributeError, так как у None нет строковых методов.

Шаги воспроизведения:
1. Создать экземпляр StringUtils
2. Вызвать метод capitalize(None)
3. Проверить результат

Ожидаемый результат: TypeError с информативным сообщением или возврат пустой строки
Фактический результат: AttributeError: 'NoneType' object has no attribute 'capitalize'

Окружение: Python 3.14.3

Возможное решение:
Добавить проверку типа в начале каждого метода:
    def capitalize(self, string: str) -> str:
        if string is None:
            return ""
        if not isinstance(string, str):
            raise TypeError(f"Expected str, got {type(string).__name__}")
        return string.capitalize()

Автор:           Svintitskiy A.

Описание:
Реализация метода contains() использует try-except для перехвата ValueError,
что менее эффективно, чем использование оператора "in".

Шаги воспроизведения:
1. Изучить код метода contains()
2. Сравнить производительность с оператором "in"

Ожидаемый результат: Использование оператора "in" для простоты и эффективности
Фактический результат: Использование try-except с index()

Окружение: Python 3.14.3

Возможное решение:
Упростить реализацию:
    def contains(self, string: str, symbol: str) -> bool:
        return symbol in string

БАГ-РЕПОРТ №6

ID:              DEF-006
Заголовок:       Метод delete_symbol() не удаляет символ, если contains() возвращает False
Приоритет:       Medium
Серьёзность:     Minor
Компонент:       StringUtils.delete_symbol()
Автор:           Svintitskiy A.
Дата:            2026-04-14

Описание:
Метод delete_symbol() использует contains() для проверки наличия символа,
что избыточно, так как replace() сам корректно обрабатывает отсутствие символа.

Шаги воспроизведения:
1. Изучить код метода delete_symbol()
2. Вызвать метод с символом, которого нет в строке

Ожидаемый результат: Прямой вызов replace() без лишней проверки
Фактический результат: Дополнительная проверка contains() перед replace()

Окружение: Python 3.14.3

Возможное решение:
Упростить реализацию:
    def delete_symbol(self, string: str, symbol: str) -> str:
        return string.replace(symbol, "")
